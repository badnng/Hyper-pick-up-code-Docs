# 问题反馈

如果您在使用澎湃记时遇到问题，或者有任何建议和想法，欢迎通过以下方式向我们反馈。

## 反馈渠道

### GitHub Issues（推荐）

我们推荐通过 GitHub Issues 提交问题反馈，这样可以更好地跟踪问题处理进度。

👉 [前往 GitHub Issues 提交反馈](https://github.com/Badnng/Hyper-pick-up-code/issues)

**提交步骤：**
1. 点击上方链接进入 Issues 页面
2. 点击 "New Issue" 按钮
3. 选择合适的 Issue 模板
4. 填写详细的问题描述
5. 点击 "Submit new issue" 提交

## 反馈内容要求

为了帮助我们更快地定位和解决问题，请在反馈中提供以下信息：

### 必要信息

1. **问题描述**
   - 清晰描述遇到的问题
   - 说明问题发生的具体场景
   - 描述期望的正确行为

2. **设备信息**
   - 手机品牌和型号
   - Android 系统版本
   - 澎湃记 App 版本号

3. **复现步骤**
   - 详细说明如何触发这个问题
   - 提供具体的操作步骤

### 辅助材料（强烈建议）

#### 取餐截图

截图可以帮助我们直观地了解问题：

**如何提供截图：**
1. 对出问题的界面进行截图
2. 在 GitHub Issue 中把图片上传

**截图注意事项：**
- 🔒 请遮挡敏感的个人信息（如手机号、地址）
- 📱 确保截图清晰，包含完整的界面信息
- 🖼️ 可以上传多张截图展示问题的不同阶段

#### 日志信息

日志可以帮助我们追踪问题的技术细节：

**如何导出日志：**

1. **方法一：App 内导出**
   - 打开澎湃记 App
   - 在底栏的'日志'页面筛选日志级别
   - 选择导出位置并保存

2. **方法二：通过 ADB 导出**（适用于开发者）
   ```bash
   adb logcat -d > hyper_pick_up_code_log.txt
   ```

**日志包含内容：**
- App 运行日志
- 错误堆栈信息
- 识别过程记录
- 系统兼容性信息

**日志隐私说明：**
- 📋 日志仅包含技术运行信息
- 🔐 不会记录您的个人数据
- 🗑️ 问题解决后可随时删除日志文件

## Issue 模板说明

我们提供了多种 Issue 模板，选择合适的模板可以让反馈更清晰：

### 🐛 Bug Report（Bug 反馈）
适用于：App 功能异常、崩溃、识别错误等问题

### 💡 Feature Request（功能建议）
适用于：希望新增功能、改进现有功能等建议

### 📝 Documentation（文档问题）
适用于：文档错误、内容缺失、需要补充说明等

### ❓ Question（问题咨询）
适用于：使用疑问、操作指导等咨询

## 反馈处理流程

1. **提交反馈** → 通过 GitHub Issues 提交
2. **确认收到** → 我们会在 1-3 天内回复确认
3. **问题分析** → 技术团队分析问题原因
4. **解决方案** → 提供修复方案或建议
5. **版本发布** → 修复将在下个版本中发布（Bug 类问题）

## 常见问题

### Q: 提交反馈后多久能得到回复？
A: 我们通常会在 1-3 个工作日内回复。复杂问题可能需要更长时间分析。

### Q: 如何查看反馈处理进度？
A: 在 GitHub Issues 中可以查看您提交的问题状态，我们会及时更新处理进展。

### Q: 能否私密反馈问题？
A: 如果涉及隐私问题，可以通过邮件联系wsj316@outlook.com，或在 Issue 中说明需要私密处理。

### Q: 如何获取最新版本？
A: 访问 [GitHub Releases](https://github.com/Badnng/Hyper-pick-up-code/releases) 下载最新版本。
